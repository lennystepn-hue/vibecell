# Impeccable Design Context ÔÇö Vibecell

This file is the persistent design brief that future design work in this
repo should respect. Synthesized 2026-04-26 from the existing codebase
(tokens.css, ProjectGridDashboard.vue, Cockpit components) and the
brand stance the founder has consistently chosen during pricing /
landing iterations.

## Design Context

### Users

- **Indie devs / vibecoders** ÔÇö work alone or in 2-3 person teams,
  cycle through 5ÔÇô30 side-projects a year, live in their terminal,
  Claude Code window, Cursor, Zed. Power users.
- They distrust generic SaaS UI: gradients with three stops, bouncy
  hero copy, stock photography, "Most popular Ô¡É" stickers, unsolicited
  AI emoji.
- Job: keep the dashboard a single-source-of-truth for what they're
  shipping; check status quickly; act decisively. Reading time per
  visit is short.

### Brand Personality

Three words: **Technical ┬À Calm ┬À Quietly confident**.

- Voice: dense, precise, never apologetic, never over-eager.
- The product itself is the demo. Marketing copy never has to oversell.
- Numbers and codepoints get prominence; sentences stay short.
- Mono is the brand voice; sans is the body voice.

### Aesthetic Direction

- **Cockpit / instrument-panel.** Dense information, fine grid, small
  labels, the user is a pilot and the dashboard is their console.
- **Glass surfaces.** Translucent panels (`var(--bg-chrome)`,
  `var(--bg-surface)`) over a dark-canvas gradient. `backdrop-filter`
  blur where surfaces stack.
- **Signal palette is semantic, not decorative.** Green = healthy /
  active / shipping. Amber = attention / launch / trial-ending. Red =
  failure / stop. Blue = neutral muted. Never use signal colors for
  pure aesthetic accent.
- **One theme done well, two themes done well.** Dark is canonical;
  Paper is a respectful daytime mode. All components must work in both.
- References that are right: Linear, Raycast, Tailscale dashboard,
  Vercel CLI output. Anti-references: Notion's marketing site, anything
  with a unicorn or ­ƒÄë in production.

### Design Principles

These five rules supersede personal taste calls in any future work.

1. **Mono labels mark sections, not headlines.** The `// what's
   included` / `// dashboard` / `//notes` pattern is the Vibecell
   signature. A new section gets a mono label above it. Never give a
   mono label a sentence ÔÇö it's a tag, not a heading.

2. **Numbers are louder than words.** When a screen has both a
   sentence and a numeric fact, the numeric fact wins the largest
   weight. Pricing pages: the price is the hero, not the verb.

3. **Emoji is a smell.** No emoji in launch ribbons, banners, error
   messages, button text. The ONLY place emoji appears in production
   UI is user-generated content (project emoji avatar ÔÇö picked by the
   user themselves) and the deliberate `Ôùê` / `Ôùç` / `Ôùó` Cockpit marks.

4. **Single accent per surface.** A card uses one signal color
   (amber for launch, green for active, red for failure). No
   multi-stop gradients across two signal colors ÔÇö each one fights for
   attention. If a launch card is amber, every accent on that card is
   amber or muted.

5. **Strikethrough is a typographic gesture.** When showing was-price
   vs is-price, the struck-through old price is **smaller, in
   fg-subtle, with a 2px line at 8% rotation**. It's a footnote to the
   real price, not a co-equal sibling. Never both prices in the same
   weight.

### Implementation tokens

- **Typography**: `Geist Sans` body, `Geist Mono` for `mono-label`
  / labels / numerals in pricing. `font-feature-settings: "ss01"`
  on mono.
- **Colors**: import from `tokens.css` only. Never hex literals in
  Vue templates unless the value is intentionally one-off (rare ÔÇö
  if you're reaching for a hex, you probably want a token).
- **Spacing**: 4px base unit. Tailwind's `gap-3` / `gap-5` / `p-5` /
  `p-7` are the canonical card sizes. Don't reach below `gap-2` unless
  you mean it.
- **Motion**: `var(--ease-out)` for 90% of transitions. `200ms` is the
  default duration. Anything bouncy is wrong.

### Anti-checklist

If a UI proposal contains any of these, stop and rework:

- [ ] An emoji that isn't a user-picked project avatar
- [ ] A multi-stop linear-gradient as a backdrop ("from-violet to-pink")
- [ ] Two prices in the same font-weight + size
- [ ] A button label longer than 4 words
- [ ] Sentence-case headlines that try to be witty ("Start free. Ship
  serious." ÔåÆ admire-but-replace)
- [ ] "Most popular" stickers / ribbons / chips
- [ ] Sales copy that addresses the user as "you" more than once per
  300 words
- [ ] A trial/launch CTA that uses a verb other than the action verb
  ("Get launch price" not "Snag your spot")
